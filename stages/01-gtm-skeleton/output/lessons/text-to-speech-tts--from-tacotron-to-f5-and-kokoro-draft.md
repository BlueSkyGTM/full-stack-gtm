# Text-to-Speech (TTS) — From Tacotron to F5 and Kokoro

## Hook
Your ear can detect a 10ms misalignment in voicing onset. TTS systems must synthesize waveforms at 24,000 samples per second with phoneme-level timing precision. This lesson traces how the field moved from hand-engineered vocoder pipelines to end-to-end generative models that clone a voice from 10 seconds of audio.

## Concept
TTS decomposes into two subproblems: (1) generate a time-aligned intermediate representation (mel spectrogram or latent sequence) from text, and (2) convert that representation into a waveform. Tacotron (2017) proved a seq2seq model with attention could replace the entire front-end linguistic pipeline. Modern systems like F5-TTS replace the attention-based decoder with flow matching, and Kokoro uses a distilled architecture for real-time inference. The mel spectrogram remains the standard intermediate representation — a compressed frequency-domain encoding that captures what the ear actually perceives.

## Mechanism
Three architectural generations, each solving a specific failure mode of the previous:

1. **Tacotron 2** (2017): Encoder-decoder with additive attention. Text → character embedding → CBHG encoder → attention context → autoregressive mel decoder → WaveNet vocoder. Failure mode: attention drift on long utterances causes word skipping or repetition. Attention is sequential — O(n) inference.

2. **Non-autoregressive models** (FastSpeech, VITS, 2020–2022): Duration predictor replaces attention. Parallel decoding of all mel frames simultaneously. VITS combines the spectrogram generator and vocoder into a single end-to-end trainable system using variational inference and normalizing flows. Failure mode: duration prediction errors cause prosody artifacts; harder to train than autoregressive models.

3. **Flow-matching and diffusion models** (F5-TTS, 2024): Treat mel spectrogram generation as a continuous normalizing flow problem. EMA (Exponential Moving Average) teacher-student distillation for inference speed. F5-TTS achieves few-shot voice cloning by conditioning on a reference audio embedding. The vocoder evolution is equally critical: WaveNet (autoregressive, slow) → WaveGlow (normalizing flow, faster) → HiFi-GAN (adversarial, fast, standard for production).

## Code
**Easy:** Generate a mel spectrogram from text using `coqui-tts` (or `piper-tts`), print the spectrogram shape, and confirm the sample rate matches the model's expected output. Save the waveform to disk, verify it plays.

**Medium:** Run inference on the same sentence with two different speaker embeddings from a multi-speaker model. Compute cosine similarity between the two resulting mel spectrograms. Print the similarity score — this quantifies how much the speaker embedding actually changes the output.

**Hard:** Implement a minimal duration predictor in PyTorch. Given a sequence of phoneme embeddings and a target mel length, learn a linear mapping that predicts frame counts per phoneme. Train on 50 utterances from a synthetic dataset (generate mels from random noise as stand-in). Print the MAE between predicted and actual durations.

## Use It
TTS productionizes personalized outreach at scale. In the Clay waterfall (GTM Zone 2 — Enrich/Engage), a contact's company name, role, and recent trigger event become input text; TTS generates a personalized voice note attached to an email or LinkedIn message. F5-TTS's few-shot cloning means the "speaker" can be your AE's actual voice, not a synthetic one.

**Redirect:** Zone 2, Enrich → Engage pipeline. Specifically: personalized async video/voice outreach where the script is generated by an LLM from enrichment data, rendered by TTS, and attached to a sequence step.

## Ship It
Production TTS requires handling the failure modes that research demos ignore:

- **Latency budgets:** Real-time factor (RTF) must be < 0.3 for interactive use. Measure it: `inference_time_seconds / audio_duration_seconds`. Print this for every inference call.
- **Text normalization:** "Dr. Smith paid $1,200 on 1/15/2024" must expand to "Doctor Smith paid one thousand two hundred dollars on January fifteenth twenty twenty four." Use a front-end normalizer (e.g., `gruut` or the TTS model's built-in) before inference. Test with 20 edge-case sentences containing abbreviations, numbers, and URLs.
- **Speaker consistency:** For voice cloning, the reference audio must be clean (no background noise, >16kHz sample rate, 5–15 seconds). Implement a pre-check: compute the reference audio's SNR estimate and reject clips below a threshold.
- **Chunking long text:** Models degrade on inputs > 500 characters. Implement sentence-level splitting with cross-sentence silence padding (150–200ms). Concatenate the resulting waveforms.
