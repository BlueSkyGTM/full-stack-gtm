**Important:** GLM reasoning models (GLM-5.1, GLM-5, GLM-5.1) require `max_tokens >= 500` or the call fails silently. All skills set `max_tokens=8192` as minimum. GLM-5-Turbo and GLM-5-Turbo are non-reasoning models; they accept lower `max_tokens` values.

## Unsupported Parameters (Z.ai)

The following OpenAI-standard parameters are NOT accepted by Z.ai and must be stripped before forwarding:
- `frequency_penalty`
- `presence_penalty`
- `logit_bias`
- `logprobs`
- `top_logprobs`
- `n` (>1 completions per request)

The operator-kit proxy layer strips these automatically. If calling Z.ai directly (not via operator-kit), strip them in the request body.

## Dispatch Pattern (Governed Maze)

Claude Code is the orchestrator. GLM engines are execution-only.

```
Claude Code reads brief → distills task-specific prompt (≤500 tokens) → blind handoff to GLM via Z.ai
GLM returns output → Claude Code validates → hands forward to next step
```

**Rules:**
- GLM never reads the repo. Claude Code extracts only what GLM needs for the specific task.
- The brief file is Claude Code's reference. It is NOT injected wholesale into GLM's system prompt.
- 500-token system prompt limit is intentional — context pollution degrades GLM output quality.
- Each handoff is blind: GLM receives task instructions + minimal spec excerpt. Nothing else.

**Why:** Sub-agents with full repo context return polluted, less focused output. The governed maze keeps each engine scoped to exactly one task. Same principle as Helix's decision tree — structured hand-off, not open conversation.

## Fallback Model

If Z.ai is unreachable or returns 5xx, the skill aborts with an error message. There is no fallback model — do not reroute to OpenAI or Anthropic. The fail-fast behavior is intentional: silent fallback produces content that looks correct but is generated by a different model with different calibration.

## Token Budget Notes

`{{GLM_BUDGET}}` is real — token spend is tracked. Lyra content skills should minimize prompt length by:
1. Writing large context to /tmp/ files and passing file paths as argv (not embedding context in the prompt string)
2. Using structured output requests ("Output ONLY a JSON array") to avoid preamble tokens
3. Keeping system prompts under 500 tokens

Hypatia and Echo should target the smallest prompt that produces a deterministic verdict or inventory.
