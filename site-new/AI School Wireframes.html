Recommendation · local-first, REST-ready

Keep today's behavior: progress lives in localStorage and can be exported/imported as JSON. Wrap every read/write in one tiny ProgressStore module. Today it talks to localStorage; the day you wire up WordPress, you swap the adapter to hit the WP REST API — no screen has to change.

Phase 1 — ship now (no backend)

• Reskin + game HUD on top of existing site
• XP / levels / badges computed from the progress object
• localStorage + JSON export/import (already built)
• Player Card reads the same object
Risk to the 800 hrs: none. Pure front-end.

Phase 2 — WordPress sync (when ready)

• Register a custom REST route on WP Personal (allowed — it's the REST API, not file uploads)
• Store progress JSON in user meta
• Swap the ProgressStore adapter → auto-saves per logged-in user
• Profile becomes a public, linkable URL for employers
