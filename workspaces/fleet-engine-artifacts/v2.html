The Code

Three layers.
Read the pipeline here.

The pipeline runs across three code layers with three different jobs. Node.js calls Vertex AI and enforces output schemas. Python ADK deploys the same agents to hosted Agent Engine and validates every GCS output with Pydantic. The agent prompts in REFERENCES.md are the third layer — version-controlled instructions that survived 1,500 cycles unchanged. Read all three here.

agents/neptune.js Node.js
Agent runtime. Each agent calls Vertex AI directly via google-auth-library. responseSchema enforcement means Neptune cannot return prose — Outreach_Bite, Neptune_Log, and funding_signal are typed fields at the API level.
agents.py Python ADK
Orchestration layer. LlmAgent definitions deploy to Google Agent Engine. Each agent gets a RagTool pointing to its Vertex AI corpus. The ADK path runs hosted — no local process needed.
REFERENCES.md → NEPTUNE_SYSTEM Agent Prompts
Instruction layer. Version-controlled, tested across 1,500 cycles, transferred verbatim from V1 to V2. The Schwartz framework, contact framing, and output contract — all written as structured natural language.

Validate

Node.js runs the agents.
Python checks every output.

The pipeline writes four typed JSON files to GCS per run. Python reads them at the boundary, validates every field with Pydantic, enforces that leads can only move forward (no stage can skip), strips Gemini citation artifacts, and validates the Node-to-Python IPC handshake. If any field is wrong, it fails loud — before anything reaches a CRM. Select a pattern to read the field notes.
