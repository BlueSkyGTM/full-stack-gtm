The Three Architecture Decisions

What the Failures Forced.

After 18 documented failures across three domains, the pattern was clear. Three classes of environmental failure demanded three architectural eliminations. V2 is what happens when you follow each one to completion.

Decision 01 — From Scraping Domain
Eliminate the scraping infrastructure
6 failures traced to data acquisition: aggregator URLs slipping through blocklists, hallucinated companies with no real URL, exclusion lists lost between server restarts, and job links that rotated before Nemo could verify them. Every patch was local. The class of failure was architectural — the scraper had no persistent state and no grounded source of truth.
↓
V2's Ahab runs via Vertex AI Grounding (Google Search). Real URLs, real companies, grounded evidence — not scraped HTML. The aggregator blocklist is enforced at the model level. Exclusion lists persist in the RAG store across runs.
Decision 02 — From Routing Domain
Eliminate the orchestration middleware
6 failures traced to n8n and Postgres: expression evaluation errors on payload shape, retry loops producing duplicate rows, connection pool exhaustion under concurrent agent load, and the single HTTP server serializing all agent calls through one process. Every fix introduced new configuration surface for the next failure.
↓
V2 uses GCS file handoff — Ahab writes ahab_output.json, Nemo reads it. No polling. No Postgres. No n8n expressions to interpret. Claude Code is the orchestrator, not a server. Sequential handoff through files is stateless and auditable.
Decision 03 — From Output QA Domain
Enforce output contracts at the model level
6 failures traced to unenforceable output expectations: CATALYST_STALE missed on non-standard funding formats, grounding redirect URLs passing as Direct_URL, citation markers appearing in the final Outreach_Bite, and Neptune returning prose when the schema wasn't enforced. V1 parsed model output with hope. Hope is not a contract.
↓
Neptune uses responseSchema enforcement — Outreach_Bite, Neptune_Log, and funding_signal are typed fields, not text to parse. Nemo's output has a formal Nemo_Enrich_Audit block with explicit ACTIVE/SHIPWRECKED status. URL sanitization and citation stripping run at the utility layer, not as afterthoughts.
The Conclusion

The Environment Was the Failure Domain.

The forensic record is unambiguous. Every one of the 18 failures was in the infrastructure beneath the agents — never in the agent prompts themselves.

// Core Diagnosis · CONTEXT.md The V1 architecture invented a shared space through Postgres queries and n8n workflows instead of using a real one. The agents were designed for sequential handoff but were forced to communicate through polling, status fields, and middleware interpretation layers.

The solution is not a better Postgres setup or a more stable Docker configuration. The solution is eliminating the execution environment as a failure domain entirely.

V2 proves this. The three agent prompts transferred verbatim. The 18 failure modes did not.
