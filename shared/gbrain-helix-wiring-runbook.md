# GBrain ↔ Helix Wiring Runbook

**For:** Stage 05 (Helix Build) and Stage 08 (Agent Wiring)  
**Purpose:** Document the exact steps taken to fix gbrain's embedding pipeline, so the wiring phase starts from a known-good state rather than re-diagnosing from scratch.

---

## What "Helix open-brain" means

Helix open-brain is **not** gstack's gbrain. It is a student-facing product feature:
- gbrain stores the student's learning history (visited lessons, quiz answers, spaced repetition state)
- FSRS (Free Spaced Repetition Scheduler) runs on top of that history to schedule card reviews
- Claude Code (or a lightweight Helix agent) reads gbrain to personalize greetings, surface weak spots, and suggest next lessons

The gstack gbrain (operator brain, this repo's content) and the Helix student brain are separate instances.

## Operator Brain — What We Fixed (2026-06-12)

### Root cause

gbrain was initialized with `openai:embedding-3` (1536d). We have no `OPENAI_API_KEY`. Z.ai's paas/v4 endpoint does not serve embeddings at all (returns 400 for every model name). Z.ai BigModel's `embedding-3` also returns 400 — the model is not activated in the Z.ai console resource packages.

All zhipu:* embedding model name variants failed:  
`embedding-3`, `embedding-2`, `embedding-v3`, `embedding_v2`, `text-embedding-v3`, `text-embedding-v2`, `GLM-Embedding`, `Embedding-3` → all 400.

### Fix applied

1. **Installed Ollama** (local inference server, free, no API key):
   ```
   winget install Ollama.Ollama --accept-package-agreements --accept-source-agreements
   ```
   Version installed: 0.30.6. Service starts automatically on login.

2. **Pulled nomic-embed-text** (274 MB embedding model, 768d):
   ```
   ollama pull nomic-embed-text
   ```

3. **Verified the provider works**:
   ```
   gbrain providers test --touchpoint embedding --model ollama:nomic-embed-text
   # ✓ 11560ms, 768 dims
   ```

4. **Re-initialized the brain** (required when switching dimensions — PGLite is schema-bound):
   ```powershell
   Move-Item "$env:USERPROFILE\.gbrain\brain.pglite" "$env:USERPROFILE\.gbrain\brain.pglite.bak" -Force
   gbrain init --pglite --embedding-model ollama:nomic-embed-text --embedding-dimensions 768
   ```

5. **Re-synced the repo**:
   ```
   gbrain sync --repo <path-to-repo>
   ```
   Result: 386 repo pages, 2763 chunks, 100% embedded. Plus 663 gstack pages auto-synced.

6. **Verified health**:
   ```
   gbrain doctor
   # [OK] embeddings: 100% coverage, 0 missing
   # [OK] embedding_provider: ollama:nomic-embed-text ✓ 124ms, 768 dims, DB aligned
   ```

## Student Brain — Initialization (Stage 05, when building Helix)

The **student brain** is a separate gbrain instance per student (stored in their fork under `progress/`). It uses the same `ollama:nomic-embed-text` provider.

Initialization spec (to be implemented in Stage 05):
```
# Student fork init (runs once on first Helix activation)
gbrain init --pglite \
  --embedding-model ollama:nomic-embed-text \
  --embedding-dimensions 768 \
  --path ./progress/brain.pglite
```

The student brain syncs:
- `progress/aifs-progress.json` → lesson completion + quiz answers
- FSRS card state (generated by Helix gate logic in Stage 05)
- Weak-spot annotations (written by `/check-understanding` post-exam)

Do NOT sync: lesson content, docs/en.md, code/ directories — those live in the operator brain.

## FSRS Integration Notes

FSRS (Free Spaced Repetition Scheduler) is the open algorithm that powers card scheduling. Key parameters:

- **Stability (S):** how long a memory lasts before dropping to 90% retention
- **Difficulty (D):** intrinsic item difficulty (1-10 scale)
- **Retrievability (R):** current recall probability (0-1)
- **Review outcome:** Again / Hard / Good / Easy (1-4 rating)

The `fsrs` block in `quiz.json` is initialized empty at content creation:
```json
"fsrs": {}
```

At runtime (Stage 05 Helix gate), Helix populates it:
```json
"fsrs": {
  "due": "2026-06-19T00:00:00Z",
  "stability": 4.2,
  "difficulty": 5.1,
  "elapsed_days": 0,
  "scheduled_days": 7,
  "reps": 1,
  "lapses": 0,
  "state": "review",
  "last_review": "2026-06-12T00:00:00Z"
}
```

The gbrain student brain stores the full card history. Helix reads it to compute `due` dates and surface review cards at the start of each session.

**Reference implementation:** `ts-fsrs` (npm) — used by most open-source FSRS implementations. Do not reimplement the algorithm; import the package.

## Key Constraint: Ollama Must Be Running

Ollama runs as a local server on `localhost:11434`. On Windows it starts automatically after install. If embeddings fail, check:

```powershell
# Check service
Get-Process ollama -ErrorAction SilentlyContinue
# If not running:
Start-Process ollama -ArgumentList "serve" -WindowStyle Hidden
```

The student-facing Helix site does NOT call ollama directly. The Helix agent (Claude Code or a lightweight script) calls gbrain, which calls ollama internally. Students never touch the embedding layer.
